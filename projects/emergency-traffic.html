Overview

Emergency response time is critical — every second counts when an ambulance is en route to a hospital. This project designs and implements a relational database system that manages traffic signal prioritization for emergency vehicles in real time.

The system uses SQL stored procedures, triggers, and views to automatically detect emergency vehicle presence and cascade signal changes across intersections to create a clear corridor. The database schema was designed using Entity-Relationship modeling to capture vehicles, routes, signals, and timestamps.

The project demonstrates advanced SQL skills including complex joins, subqueries, window functions, and procedural SQL to solve a real-world urban management problem.
