Overview

This project builds a complete supply chain analytics system — from raw data generation through ETL loading, SQL analysis, AI-powered reporting, and an interactive Tableau BI dashboard — covering the full workflow expected of a senior data analyst.

The dataset simulates 10,000 orders spanning 2022–2024 across 12 global suppliers, 20 product categories, and 30 enterprise customers. The pipeline answers critical business questions: which suppliers are creating delivery risk, which products drive the most margin, and which regions are underperforming.

The AI reporting layer uses Claude to transform live Snowflake KPIs into weekly executive summaries and anomaly alerts — making insights accessible to non-technical stakeholders without manual report writing.

Architecture

How It
Works

1. Data Generation & ETL

Python generates 10,000 realistic orders with supplier-dependent delivery delays, discount patterns, and seasonal variation. A dedicated ETL script loads all 5 tables into Snowflake with schema creation and view setup in a single command.

2. SQL Analytics Layer

Production-grade SQL including CTEs, window functions (LAG, RANK, running totals), multi-table JOINs, and CASE expressions. Six analytical views power both the Streamlit app and Tableau dashboard without duplicating logic.

3. Claude AI Reports

Fetches live KPI snapshots from Snowflake and sends them to Claude API to generate structured weekly intelligence reports and anomaly alerts. Reports are saved as Markdown files and rendered directly in the Streamlit app.

4. Streamlit Dashboard

5-page interactive reporting app: Executive Overview, Product Performance, Supplier Scorecard, Logistics & Carriers, and the AI Report generator. Built with Plotly for rich interactive charts.

5. Tableau BI Dashboard

Pre-built Tableau workbook (.twb) with 6 sheets and an executive dashboard layout — Revenue Trend, Top Products, Supplier On-Time Rate, Regional Revenue, Carrier Performance, and Revenue vs Margin scatter. Opens directly in Tableau Desktop with Snowflake connection.

6. Pipeline Orchestration

Single-command runner executes the full pipeline in sequence: data generation → Snowflake ETL → AI report → Streamlit launch. Individual steps can be run independently for development and debugging.

SQL Depth

The analytical SQL layer demonstrates patterns expected in production data engineering: month-over-month growth using LAG window functions, supplier risk scoring with conditional aggregation, customer ranking with RANK() PARTITION BY, and 30-day rolling averages using ROWS BETWEEN window frames.

Six Snowflake views encapsulate business logic — VW_MONTHLY_REVENUE, VW_PRODUCT_PERFORMANCE, VW_SUPPLIER_SCORECARD, VW_REGIONAL_SUMMARY, VW_CARRIER_PERFORMANCE, and VW_ORDER_FULFILLMENT — all acting as the single source of truth for both the BI tools and the Python application layer.

Features

Key
Highlights

Supplier Risk Scoring

Automated risk classification for all 12 suppliers based on on-time delivery rates — flagging HIGH (below 85%), MEDIUM (85–92%), and LOW risk. Critical for procurement decisions.

AI Executive Reports

Claude generates fully structured weekly business reports with executive summary, revenue analysis, supplier risk assessment, carrier evaluation, and strategic recommendations — directly from live data.

Tableau-Ready Architecture

SQL views are designed as Tableau data sources. The included .twb workbook connects directly to Snowflake — open it in Tableau Desktop, enter credentials, and the dashboard is ready instantly.
